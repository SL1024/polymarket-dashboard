## 🔌 API 配置

### 已配置的 API 端点

项目已经配置好以下 Polymarket API：

1. **交易历史 API**
   ```
   https://data-api.polymarket.com/trades?user={address}
   ```

2. **持仓价值 API**
   ```
   https://data-api.polymarket.com/value?user={address}
   ```

3. **市场信息 API**
   ```
   https://data-api.polymarket.com/markets?user={address}
   ```

## 📊 数据统计说明

### 1. 交易次数 (txCount)
- **来源**：trades API 返回的数组长度
- **计算**：直接统计交易记录数量
- **更新**：每次刷新时重新计算

### 2. 交易金额 (totalVolume)
- **来源**：trades API 中的 size 和 price 字段
- **计算**：sum(size × price)
- **单位**：美元 ($)
- **格式**：保留两位小数

### 3. 盈亏金额 (pnl)
- **来源**：trades API 中的 pnl 字段
- **计算**：sum(pnl)
- **单位**：美元 ($)
- **显示**：正数绿色，负数红色

### 4. 账户金额 (balance)
- **来源**：value API 返回的 value 字段
- **说明**：当前持仓的总价值
- **单位**：美元 ($)
- **更新**：每次刷新时获取最新值

### 5. 板块数量 (categories)
- **来源**：markets API 中的 category 字段
- **计算**：使用 Set 去重统计
- **说明**：参与的不同市场分类数量
- **示例**：Politics, Sports, Entertainment

### 6. 天活跃 (dayActive)
- **计算**：最近 24 小时内的交易次数
- **时间范围**：当前时间 - 24小时

### 7. 周活跃 (weekActive)
- **计算**：最近 7 天内的交易次数
- **时间范围**：当前时间 - 7天

### 8. 月活跃 (monthActive)
- **计算**：最近 30 天内的交易次数
- **时间范围**：当前时间 - 30天

### 9. 最后交易 (lastTxDays)
- **计算**：当前日期 - 最后交易日期
- **单位**：天
- **显示**：例如 "25天"
- **自动更新**：每天自动重新计算

## 📖 使用流程

### 首次使用

1. **打开网站**
   - 访问你的 GitHub Pages 地址
   - 或本地打开 index.html

2. **添加地址**
   - 点击"批量添加地址"
   - 输入 Polymarket 地址（每行一个）
   - 点击"确认添加"

3. **等待加载**
   - 系统自动获取数据
   - 显示"加载中..."
   - 通常需要几秒到几十秒

4. **查看数据**
   - 数据显示在表格中
   - 可以查看所有统计信息

### 日常使用

1. **刷新数据**
   - 单个刷新：点击行末的"刷新"按钮
   - 批量刷新：勾选地址 → 点击"刷新选中"

2. **管理地址**
   - 添加：点击"批量添加地址"
   - 删除：勾选地址 → 点击"删除选中"

3. **查看详情**
   - 悬停在地址上查看完整地址
   - 观察盈亏颜色（绿色盈利/红色亏损）

## 🔍 故障排查

### 问题 1：网站无法访问
- 检查 GitHub Pages 是否启用
- 等待 5-10 分钟让部署完成
- 确认仓库是 Public（公开）
- 清除浏览器缓存

### 问题 2：数据不显示
- 检查地址格式（0x开头，42个字符）
- 按 F12 查看控制台错误
- 确认地址是否有交易记录
- 检查网络连接

### 问题 3：数据一直"加载中..."
- 可能是 API 暂时不可用
- 可能是地址无效
- 可能是网络问题
- 尝试刷新页面或稍后再试

### 问题 4：盈亏显示为 0
- 这是正常现象
- API 可能不总是提供盈亏数据
- 不影响其他数据的准确性
