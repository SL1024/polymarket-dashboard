# ⚡ 快速开始指南

这是一份 5 分钟快速上手指南，让你立即开始使用这个工具！

## 🎯 目标

完成这个指南后，你将：
- ✅ 成功部署网站到互联网
- ✅ 添加第一个 Polymarket 地址
- ✅ 查看统计数据

## 📋 准备工作

你需要：
1. ✅ 一个 GitHub 账号（没有？[点这里注册](https://github.com/signup)）
2. ✅ 这个项目的所有文件
3. ✅ 至少一个 Polymarket 地址

## 🚀 三步部署

### 第 1 步：创建仓库（1 分钟）

1. 登录 [GitHub](https://github.com)
2. 点击右上角 **"+"** → **"New repository"**
3. 仓库名称输入：`polymarket-tracker`
4. 选择 **"Public"**（公开）
5. 点击 **"Create repository"**

### 第 2 步：上传文件（2 分钟）

1. 在新仓库页面，点击 **"uploading an existing file"**
2. 拖拽以下文件到上传区：
   - ✅ index.html
   - ✅ style.css
   - ✅ app.js
   - ✅ README.md
   - ✅ .gitignore
3. 点击 **"Commit changes"**

### 第 3 步：启用网站（1 分钟）

1. 点击 **"Settings"** 标签
2. 左侧菜单找到 **"Pages"**
3. Source 选择 **"main"** 分支
4. 点击 **"Save"**
5. 等待 2-3 分钟

✅ 完成！你的网站地址是：
```
https://你的用户名.github.io/polymarket-tracker/
```

## 📝 开始使用

### 第 1 步：打开网站

在浏览器中访问你的网站地址。

### 第 2 步：添加地址

1. 点击 **"批量添加地址"**
2. 输入 Polymarket 地址（每行一个）
3. 点击 **"确认添加"**

**示例地址**（用于测试）：
```
0x0000000000000000000000000000000000000001
```

### 第 3 步：查看数据

等待几秒钟，数据会自动显示在表格中！

## ❓ 遇到问题？

### 网站无法访问

- 等待 5-10 分钟让 GitHub Pages 部署完成
- 检查 Settings > Pages 是否正确配置
- 确保仓库是 Public（公开）

### 数据不显示

- 检查地址格式是否正确（0x开头，42个字符）
- 按 F12 打开开发者工具查看错误
- 确认地址是否有交易记录

### 其他问题

查看完整文档：
- [README.md](./README.md) - 项目说明
- [部署指南.md](./部署指南.md) - 详细部署教程
- [使用教程.md](./使用教程.md) - 详细使用说明
