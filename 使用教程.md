# 📚 Polymarket 账户统计工具 - 使用教程

这是一份详细的使用教程，帮助你快速上手这个工具。

## 🎯 工具概述

这个工具可以帮你：
- 追踪多个 Polymarket 账户的交易数据
- 统计交易次数、金额、盈亏
- 分析账户活跃度
- 查看参与的市场板块

## 📝 第一次使用

### 步骤 1：打开网站

在浏览器中输入你的 GitHub Pages 地址：
```
https://你的用户名.github.io/polymarket-tracker/
```

### 步骤 2：添加第一个地址

1. 点击页面顶部的 **"批量添加地址"** 按钮
2. 会弹出一个输入框
3. 输入一个或多个 Polymarket 地址（每行一个）
4. 点击 **"确认添加"**

**示例**：
```
0x1234567890123456789012345678901234567890
0xabcdefabcdefabcdefabcdefabcdefabcdefabcd
```

### 步骤 3：等待数据加载

- 添加地址后，系统会自动开始获取数据
- 你会看到 "加载中..." 的提示
- 通常需要几秒钟到几十秒

### 步骤 4：查看统计数据

数据加载完成后，你会看到一个表格，显示：

| 列名 | 含义 | 示例 |
|------|------|------|
| 序号 | 地址的序号 | 1, 2, 3... |
| 地址 | 钱包地址（简化显示） | 0x1234...7890 |
| 交易次数 | 总共进行了多少次交易 | 150 |
| 交易金额 | 所有交易的总金额 | $12,345.67 |
| 盈亏金额 | 赚了多少或亏了多少 | $+234.56 或 $-123.45 |
| 账户金额 | 当前持仓价值 | $5,678.90 |
| 板块数量 | 参与了多少个不同板块 | 5 |
| 天活跃 | 今天交易了几次 | 3 |
| 周活跃 | 本周交易了几次 | 15 |
| 月活跃 | 本月交易了几次 | 45 |
| 最后交易 | 距离上次交易多少天 | 5天 |

## 🔄 日常使用

### 刷新单个地址的数据

1. 找到要刷新的地址所在的行
2. 点击该行最右侧的 **"刷新"** 按钮
3. 等待数据更新

**使用场景**：
- 想查看某个地址的最新数据
- 发现某个地址的数据不准确

### 批量刷新多个地址

1. 勾选要刷新的地址（点击每行最左侧的复选框）
2. 点击顶部的 **"刷新选中"** 按钮
3. 等待所有选中地址的数据更新

**使用场景**：
- 每天早上查看所有地址的最新情况
- 定期更新所有数据

### 全选所有地址

1. 点击表格标题行最左侧的复选框
2. 所有地址会被自动选中
3. 可以一次性刷新或删除所有地址

### 删除不需要的地址

1. 勾选要删除的地址
2. 点击顶部的 **"删除选中"** 按钮
3. 在确认对话框中点击 **"确定"**
4. 地址会被永久删除

**注意**：删除后无法恢复，请谨慎操作！

## 💡 使用技巧

### 技巧 1：批量添加地址

你可以一次性添加多个地址：

```
0x1111111111111111111111111111111111111111
0x2222222222222222222222222222222222222222
0x3333333333333333333333333333333333333333
0x4444444444444444444444444444444444444444
0x5555555555555555555555555555555555555555
```

只需要在输入框中每行输入一个地址即可。

### 技巧 2：查看完整地址

- 鼠标悬停在简化的地址上
- 会显示完整的地址
- 可以复制完整地址

### 技巧 3：识别盈亏

- **绿色数字**：表示盈利
- **红色数字**：表示亏损
- 一目了然地看出哪些地址赚钱了

### 技巧 4：定期刷新

建议：
- **每天刷新一次**：查看最新数据
- **交易后立即刷新**：确认交易是否记录
- **不要频繁刷新**：避免被 API 限流

### 技巧 5：备份地址列表

由于数据存储在浏览器本地，建议：
1. 定期截图保存地址列表
2. 或者将地址复制到文本文件保存
3. 避免清除浏览器数据导致丢失

## 📊 数据解读

### 交易次数

- **高交易次数**：说明该地址很活跃
- **低交易次数**：可能是新用户或不活跃用户

### 交易金额

- 显示的是所有交易的总金额
- 不等于盈亏，只是交易规模

### 盈亏金额

- **正数（绿色）**：总体盈利
- **负数（红色）**：总体亏损
- **接近 0**：基本持平

### 账户金额

- 显示当前持仓的总价值
- 不包括已经结算的仓位
- 实时反映账户状态

### 板块数量

- 数字越大，说明参与的市场类型越多
- 例如：政治、体育、娱乐等

### 活跃度（天/周/月）

- **天活跃**：今天是否有交易
- **周活跃**：本周交易频率
- **月活跃**：本月交易频率

可以用来判断：
- 哪些地址最活跃
- 交易习惯和模式

### 最后交易时间

- 显示距离上次交易的天数
- **0天**：今天有交易
- **1天**：昨天最后一次交易
- **数字越大**：越久没有交易

## ⚠️ 注意事项

### 数据准确性

- 数据来自 Polymarket 官方 API
- 可能有几分钟的延迟
- 如果发现数据不准，可以刷新重试

### 数据存储

- 所有数据存储在浏览器本地
- 不会上传到任何服务器
- 只有你能看到你添加的地址

### 数据丢失

以下操作会导致数据丢失：
- ❌ 清除浏览器缓存
- ❌ 清除浏览器数据
- ❌ 使用隐私模式/无痕模式
- ❌ 卸载浏览器

### API 限制

- 不要频繁刷新（建议间隔至少 1 分钟）
- 如果遇到 429 错误，说明请求过于频繁
- 等待几分钟后再试

### 浏览器兼容性

推荐使用：
- ✅ Chrome（推荐）
- ✅ Edge
- ✅ Firefox
- ✅ Safari

不推荐：
- ❌ IE（不支持）
- ❌ 过旧的浏览器版本

## 🔍 故障排查

### 问题 1：数据一直显示"加载中..."

**可能原因**：
- 网络连接问题
- API 暂时不可用
- 地址无效或没有交易记录

**解决方法**：
1. 检查网络连接
2. 按 F12 打开开发者工具，查看 Console 是否有错误
3. 尝试刷新页面
4. 确认地址是否正确

### 问题 2：盈亏数据显示为 0

**可能原因**：
- API 没有返回盈亏数据
- 该地址确实没有盈亏

**解决方法**：
- 这是正常现象
- Polymarket API 可能不总是提供盈亏数据

### 问题 3：板块数量显示为 0

**可能原因**：
- API 没有返回市场分类信息
- 该地址参与的市场没有分类

**解决方法**：
- 这是正常现象
- 不影响其他数据的准确性

### 问题 4：最后交易时间不更新

**可能原因**：
- 该地址确实没有新交易
- 需要手动刷新数据

**解决方法**：
- 点击"刷新"按钮更新数据
- 最后交易天数会每天自动增加
