# 📦 GitHub Pages 部署指南

这是一份完整的、适合编程小白的部署教程。按照步骤操作，你就能把这个网站发布到互联网上！

## 🎯 部署前准备

你需要：
1. 一个 GitHub 账号（免费注册：https://github.com/signup）
2. 这个项目的所有文件
3. 一个浏览器（Chrome、Edge、Firefox 都可以）

## 📝 详细部署步骤

### 第一步：注册/登录 GitHub

1. 打开浏览器，访问 https://github.com
2. 如果没有账号，点击右上角 "Sign up" 注册
3. 如果有账号，点击 "Sign in" 登录

### 第二步：创建新仓库

1. 登录后，点击页面右上角的 **"+"** 号
2. 在下拉菜单中选择 **"New repository"**（新建仓库）
3. 填写仓库信息：
   - **Repository name**（仓库名称）：输入 `polymarket-tracker`（或其他你喜欢的名字）
   - **Description**（描述）：可选，例如 "Polymarket 账户交易统计工具"
   - **Public/Private**：选择 **"Public"**（公开）
   - **不要勾选** "Add a README file"
4. 点击底部绿色按钮 **"Create repository"**

### 第三步：上传项目文件

创建仓库后，你会看到一个空白页面。按以下步骤上传文件：

#### 方法一：网页上传（推荐新手）

1. 在仓库页面，找到并点击 **"uploading an existing file"** 链接
2. 将以下文件拖拽到上传区域：
   ```
   ✅ index.html
   ✅ style.css
   ✅ app.js
   ✅ README.md
   ✅ API配置说明.md
   ✅ 部署指南.md
   ✅ .gitignore
   ```
3. 在页面底部的 "Commit changes" 区域：
   - 第一个输入框会自动填写 "Add files via upload"
   - 保持默认即可
4. 点击绿色按钮 **"Commit changes"**
5. 等待上传完成（几秒钟）

#### 方法二：使用 Git 命令行（适合有经验的用户）

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/你的用户名/polymarket-tracker.git
git push -u origin main
```

### 第四步：启用 GitHub Pages

1. 在仓库页面，点击顶部的 **"Settings"**（设置）标签
2. 在左侧菜单中，向下滚动找到 **"Pages"** 选项并点击
3. 在 "Build and deployment" 部分：
   - **Source**：选择 **"Deploy from a branch"**
   - **Branch**：选择 **"main"** 分支
   - **Folder**：选择 **"/ (root)"**
4. 点击 **"Save"** 按钮
5. 页面会刷新，顶部会显示一个蓝色提示框

### 第五步：等待部署完成

1. 等待 1-3 分钟（GitHub 需要时间构建网站）
2. 刷新页面，你会看到一个绿色提示框，显示：
   ```
   ✅ Your site is live at https://你的用户名.github.io/polymarket-tracker/
   ```
3. 点击这个链接，就能访问你的网站了！

## 🎉 部署成功！

恭喜！你的网站现在已经在线了。你可以：
- 把网址分享给任何人
- 在任何设备上访问
- 随时更新内容

## 🔄 如何更新网站

如果你修改了代码，想要更新网站：

### 方法一：网页更新

1. 进入你的仓库页面
2. 点击要修改的文件（例如 `app.js`）
3. 点击右上角的铅笔图标 ✏️（Edit this file）
4. 修改内容
5. 点击底部 **"Commit changes"**
6. 等待 1-2 分钟，网站会自动更新

### 方法二：重新上传

1. 进入仓库页面
2. 点击要替换的文件
3. 点击右上角的垃圾桶图标 🗑️ 删除旧文件
4. 点击 "Add file" > "Upload files" 上传新文件
5. 等待自动部署

## 📱 使用你的网站

### 添加 Polymarket 地址

1. 打开你的网站
2. 点击 **"批量添加地址"** 按钮
3. 输入 Polymarket 地址（每行一个），例如：
   ```
   0x1234567890123456789012345678901234567890
   0xabcdefabcdefabcdefabcdefabcdefabcdefabcd
   ```
4. 点击 **"确认添加"**
5. 网站会自动获取数据并显示

### 刷新数据

- **单个刷新**：点击每行右侧的 "刷新" 按钮
- **批量刷新**：勾选多个地址，点击顶部 "刷新选中" 按钮

### 删除地址

1. 勾选要删除的地址
2. 点击 **"删除选中"** 按钮
3. 确认删除

## ⚠️ 常见问题

### 1. 网站显示 404 错误

**原因**：GitHub Pages 还没有部署完成

**解决方法**：
- 等待 5-10 分钟
- 检查 Settings > Pages 是否正确配置
- 确保选择了 "main" 分支

### 2. 数据不显示

**原因**：可能是 API 问题或地址无效

**解决方法**：
- 按 F12 打开开发者工具，查看 Console 是否有错误
- 确认输入的是有效的以太坊地址
- 检查地址是否有交易记录

### 3. 网站样式错乱

**原因**：文件路径问题

**解决方法**：
- 确保所有文件都在仓库根目录
- 检查 `index.html` 中的文件引用路径
- 清除浏览器缓存后重试

### 4. 无法上传文件

**原因**：文件太大或格式不支持

**解决方法**：
- 确保单个文件小于 25MB
- 检查文件名是否包含特殊字符
- 尝试分批上传

## 🔒 数据安全说明

- ✅ 所有数据存储在你的浏览器本地（localStorage）
- ✅ 不会上传到任何服务器
- ✅ 只有你能看到你添加的地址
- ⚠️ 清除浏览器数据会导致记录丢失
- 💡 建议定期备份地址列表

## 🎓 进阶技巧

### 自定义域名

如果你有自己的域名，可以：

1. 在 Settings > Pages 中找到 "Custom domain"
2. 输入你的域名（例如：tracker.yourdomain.com）
3. 在域名提供商处添加 CNAME 记录
4. 等待 DNS 生效（可能需要几小时）

### 启用 HTTPS

GitHub Pages 自动提供 HTTPS，但如果使用自定义域名：

1. 在 Settings > Pages 中
2. 勾选 "Enforce HTTPS"
3. 等待证书生成（可能需要几分钟）
